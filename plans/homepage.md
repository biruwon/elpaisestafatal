# Homepage Specification v1.0

## Status

Approved

---

# Goal

The homepage should answer one question:

> "Is Spain really doing as badly as people say?"

The homepage should not try to explain every issue.

Instead, it should encourage users to start exploring Spain's current concerns through the most discussed public claims.

---

# Success Criteria

Within 30 seconds, a first-time visitor should:

- Understand what the website is about.
- Discover at least one current controversial claim.
- Understand that every claim is backed by data.
- Continue into a claim page or one of the main concerns.

---

# Information Hierarchy

The homepage should contain the following sections in this order.

## 1. Hero

Purpose:

Explain the mission of the website.

Current direction:

España está fatal.

¿O no?

Descubre qué dicen realmente los datos.

The hero should immediately communicate that the website investigates whether common public beliefs about Spain are supported by evidence.

---

## 2. Search

Purpose:

Allow users to quickly navigate to a claim or concern.

This is NOT the primary feature in the MVP.

Requirements:

- Simple search box.
- Autocomplete suggestions.
- Suggestions should initially come from existing claims.
- As the user types, matching claims appear.
- Empty searches should display popular claims.

Future versions may support:

- URLs
- Tweets
- Images
- Videos
- AI-generated answers

Not included in MVP.

---

## 3. Trending Claims

Purpose:

Provide the main entry point into the website.

This is the most important homepage section.

Display:

5–10 current claims.

Examples:

- "La inmigración aumenta la delincuencia"
- "España tiene demasiados impuestos"
- "Los alquileres suben por la inmigración"

Each card should contain:

- claim
- concern
- last updated
- short subtitle
- link to claim page

The homepage should encourage users to investigate current discussions rather than search from scratch.

---

## 4. Main Concerns

Purpose:

Allow users to browse investigations.

Display the existing concerns.

Example:

🏠 Vivienda

🌍 Inmigración

🏛 Política

💰 Economía

...

Each concern links to its topic page.

---

# Removed from MVP

The homepage should NOT include:

- Newsletter
- Statistics dashboard
- Latest investigations
- Latest updates
- Methodology
- Source lists
- Random claims
- User accounts
- Personalization

These may appear elsewhere.

---

# Visual Tone

The homepage should feel:

- Friendly
- Calm
- Curious
- Non-partisan

Avoid:

- Alarmism
- Activism
- Government branding
- Academic complexity

The goal is to invite exploration.

---

# User Journey

Expected flow:

Homepage

↓

Trending Claim

↓

Claim Page

↓

Related Claims

↓

Topic Investigation
