# Visual Design System Specification v1.0

## Status

Approved

---

# Goal

The Visual Design System defines how El País Está Fatal should look and feel.

Its purpose is not only to create a visually consistent website, but to make complex public issues easier to understand through thoughtful design.

Every visual decision should help users:

* Understand information faster.
* Navigate naturally.
* Focus on evidence.
* Feel curious rather than overwhelmed.

The design should reinforce the identity of the project as a modern, evidence-based explainer about Spain.

---

# Brand Personality

El País Está Fatal should feel like a combination of:

* Modern explainer.
* Data storytelling platform.

It should **not** feel like:

* A traditional newspaper.
* A government website.
* An academic paper.
* A fact-checking organisation.

The visual identity should communicate:

* Curiosity.
* Clarity.
* Trust.
* Simplicity.
* Evidence.

Never:

* Alarmism.
* Aggression.
* Partisanship.
* Sensationalism.

---

# Design Principles

Every page should follow these principles:

## Visual before text

Important ideas should appear as visuals before long explanations.

## One idea per section

Each section should communicate one concept.

Avoid mixing multiple unrelated ideas.

## Easy to scan

Users should understand the structure of a page in seconds.

## Mobile-first

Design starts with mobile.

Desktop expands the experience rather than defining it.

## White space is intentional

The interface should breathe.

Avoid dense layouts.

## Evidence first

Visuals support evidence.

Decorative elements should never compete with charts or explanations.

---

# Information Density

The website should combine:

* The clarity of Apple.
* The evidence-driven style of Our World in Data.
* The visual consumption habits of modern social platforms.

Avoid:

* Long uninterrupted paragraphs.
* Walls of text.
* Crowded dashboards.
* Excessive decoration.

---

# Typography

Typography should prioritise readability.

Guidelines:

* Large page titles.
* Clear section hierarchy.
* Short paragraphs.
* Frequent lists.
* Plenty of spacing.
* Limited line length.

Avoid:

* Long blocks of text.
* Tiny captions.
* Multiple font families.
* Decorative typography.

Typography should support scanning before reading.

---

# Colour System

Colours identify topics.

They do **not** indicate whether something is good, bad or severe.

Each topic receives a unique colour used consistently across:

* Homepage.
* Topic Pages.
* Claim Pages.
* Cards.
* Charts.
* Icons.
* Shareable assets.

The colour palette should remain calm and professional.

---

# Iconography

Every topic should have a unique outline icon.

Icons should be:

* Simple.
* Minimal.
* Easily recognisable.
* Consistent in stroke width.
* Readable at small sizes.

Avoid:

* Emojis.
* Filled cartoon icons.
* 3D icons.
* Highly detailed illustrations.

Icons help users recognise topics quickly without distracting from the content.

---

# Illustrations

Each topic should include a custom AI-generated illustration.

Purpose:

* Give personality to the topic.
* Improve recognition.
* Make pages more visually engaging.

Illustrations should:

* Be consistent in style.
* Remain secondary to the data.
* Avoid political symbolism.
* Avoid emotional manipulation.
* Never replace evidence.

Illustrations are decorative.

Charts communicate the facts.

---

# Charts

Charts are one of the defining elements of El País Está Fatal.

Every important chart should answer a question.

Each chart should follow this structure:

Question

↓

Key takeaway

↓

Chart

↓

Short explanation

Charts should never appear without context.

The chart itself is not the message.

The insight is the message.

---

# Visualisations

Preferred visualisations:

* Line charts.
* Bar charts.
* Area charts.
* Maps.
* Timelines.
* Comparison tables.
* Infographics.

The choice depends on the data being explained.

Avoid using unnecessary visualisations.

Every visual should exist to answer a specific question.

---

# Cards

The website should use a consistent card system.

Different card types may exist, but they should belong to the same visual family.

Shared characteristics include:

* Consistent spacing.
* Rounded corners.
* Similar typography.
* Consistent shadows.
* Shared button styles.
* Shared interaction patterns.

Different content types should remain visually distinguishable while still feeling like part of the same product.

The detailed Knowledge Card system will be defined separately.

---

# Motion

The MVP should use minimal motion.

Animations should:

* Feel natural.
* Improve understanding.
* Never distract.

Examples:

* Small hover effects.
* Smooth page transitions.
* Expand/collapse sections.

Avoid:

* Decorative animations.
* Automatic movement.
* Excessive scrolling effects.

---

# Mobile Experience

Mobile is the primary experience.

The website should not simply shrink the desktop layout.

Instead:

* Reorganise content for vertical reading.
* Keep all information available.
* Optimise touch interactions.
* Prioritise readability.
* Maintain generous spacing.

Desktop layouts should extend the mobile experience rather than redefine it.

---

# Sharing

The visual language should support sharing.

Every important card should be capable of becoming a shareable asset.

Shared visuals should:

* Be understandable independently.
* Include subtle branding.
* Remain visually attractive.
* Encourage curiosity.

Sharing should feel like a natural extension of reading.

---

# Accessibility

The interface should be usable by the widest possible audience.

Guidelines:

* High colour contrast.
* Large touch targets.
* Keyboard accessibility.
* Meaningful alternative text.
* Readable typography.
* Clear focus states.

Accessibility should be considered during implementation rather than added afterwards.

---

# Visual Consistency

Every page should immediately feel like part of El País Está Fatal.

Consistency should be achieved through:

* Colours.
* Typography.
* Icons.
* Card styles.
* Illustrations.
* Charts.
* Layout spacing.

Users should recognise screenshots of the website even without seeing the logo.

---

# Future Evolution

The design system should support future formats without requiring major redesign.

Examples:

* Instagram carousels.
* Vertical videos.
* Presentation slides.
* Printable reports.
* Interactive explainers.
* Podcasts with accompanying visuals.

The same visual language should extend naturally across all future formats.

---

# Success Criteria

The Visual Design System succeeds when:

* Users understand information faster.
* The website feels modern and trustworthy.
* Every page is visually consistent.
* Evidence becomes the centre of the experience.
* Complex topics become easier to explore.
* Shared visuals are immediately recognisable as part of El País Está Fatal.
